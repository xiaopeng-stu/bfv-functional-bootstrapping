# Algorithm-to-code mapping

This note records how the paper-level construction is represented in `main.go`.

## Paper Step 1: homomorphic decryption and sparse packing

Paper task: given LWE ciphertexts

```text
b_i = <a_i, s> + alpha * mu_i + e_i mod q,
```

homomorphically compute and sparsely pack

```text
1_r ⊗ (alpha*mu_0+e_0, ..., alpha*mu_{m-1}+e_{m-1}).
```

Main code entry points:

- `planLWEToRLWEStep1`
- `generateRandomLWECiphertexts`
- `preprocessEncryptedSecretBlocks`
- `homomorphicDecryptLWEToSparseRLWE`
- `homomorphicPackLWECiphertexts`
- `sparseRotateAndSumStep1`

The benchmark summary calls this stage `LWE->RLWE Step1`.

## Paper Algorithms 3, 4, and 5: sparse polynomial evaluation

Paper task: evaluate LUT polynomials on a sparse BFV ciphertext with repetition multiplicity `r = N/m`.

Main code entry points:

- `MonomialGenExtra` and `MonomialGenExtraPrecomp`: generate packed powers.
- `buildPatersonStockmeyerMatrices`: build the Algorithm-5 block coefficient matrices.
- `preprocessParallelLT3ViewsAtLevel`: prepare BatchLT plaintext masks.
- `parallelLT3BSGSHoistedPrecomp`: hoisted BSGS implementation of the BatchLT step.
- `polyEvalSparsePow2Alg5Precomp`: precomputed Algorithm-5 polynomial evaluation path.
- `polyEvalSparsePow2Alg5`: non-precomputed Algorithm-5 path.
- `benchPolyEvalWithPlan`: benchmark-facing dispatcher.
- `sparseRotateAndSum`: final row-wise summation.

The benchmark summary splits this stage into:

- `Step 2 EvalPower`
- `Step 2 BatchLT`
- `Step 2 Total`

## Degree handling

The program supports two related cases.

- `degree + 1 = 2^k`: direct Algorithm-5 path.
- `degree = 2^k`: split-top wrapper. The lower degree-`degree-1` part is evaluated by Algorithm 5, and the top term is added separately.

The 9-bit benchmark uses `degree = 65536`, which triggers the split-top case.

## BFV/RLWE-to-LWE tail

Paper task: convert the BFV output back to LWE ciphertexts and switch from the internal Q-prime modulus to the target modulus `T`.

Main code entry points:

- `tailTauInvariantTargetCoeffs`
- `tailExtractionPositions`
- `tailManualFinalKeySwitchNoSpecialQPrime`
- `tailSampleExtractMany`
- `tailLWEModSwitch`
- `tailComputeNoiseStats`
- `printFailureProbabilitySummary`

For `m = 1`, the program skips sparse SlotToCoeff by default because the polynomial-evaluation output is already a replicated constant plaintext. This is controlled by `-skip-stc-m1=true`.

## Diagnostics

Useful diagnostics for checking level consumption and noise behavior:

- `-mul-trace`: operation-level multiplication and rotation trace.
- `-mul-trace-noise`: decrypt after logged operations to fill noise columns. This is slow.
- `-poly-noise-trace`: exact polynomial-evaluation noise trace.
- `-ct2-noise-probe`: immediate grouped-powers checkpoint.
- `-qprime-ks-noise`: Q-prime-domain noise before/after final key switching.
- `-output-noise-table`: final per-output LWE noise table.

## Data files

- `data.txt` is the cleaned UTF-8 PowerShell log used to generate the current paper-style tables.
- `summary.csv` is generated by `scripts/parse_summary.py`.
- `docs/raw/run_20260704_013451.zip` stores the original uploaded raw logs.
- The current benchmark data use a sparse ternary LWE secret with Hamming weight `h=512`.
